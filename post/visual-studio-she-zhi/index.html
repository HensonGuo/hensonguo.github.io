Visual Studio中VC++目录与C/C++及链接器的包含目录、库目录的区别

分析

最初的VC++目录对应于Microsoft Windows的SDK，它的变量也是Windows的环境变量（PATH），而C/C++中的设置不是设置环境变量，它是设置命令行参数的，只针对当前工程。即：“VC++目录”影响全局，只要是电脑上这个VS创建的工程，都会被这个设置影响，而C/C++只影响单个工程。

区别

但是，在VS2010之后，VC++目录也只会影响单个工程了。之后版本VS的主要区别：VC++目录是一个Windows环境变量，C/C++是命令行参数，这是本质区别；相同的项，VC++目录中的包含目录对应C/C++中的附加包含目录，VC++目录中的库目录对应链接器中的附加库目录，二者效果相同。

项目属性（.props配置文件）

.props 文件是 Visual Studio 中的一种工程属性表文件（project property sheet），用于定义工程的编译和链接选项，包括编译器选项、链接器选项、附加包含目录、附加库目录等。通过使用 .props 文件，可以方便地在多个工程中共享相同的编译和链接选项，提高工作效率。

方法：
1：打开属性管理器，视图->其他窗口->属性管理器
2：在属性管理器视图中选中项目->右键->添加属性表


VS 点击文件自动定位到解决方案资源管理器中文件所在目录位置

Tools->Option->Projects And Solutions->General, tick “track Active Item in Solution Explorer”
